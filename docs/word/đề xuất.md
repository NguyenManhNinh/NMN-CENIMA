PHỤ LỤC SẢN PHẨM ĐỒ ÁN TỐT NGHIỆP
1.	Sản phẩm phải chạy được và demo được
-	Website / ứng dụng / mô hình AI hoạt động bình thường.
-	Chạy được trên localhost hoặc hosting khi bảo vệ.
-	Mọi chức năng chính phải thực thi được trong buổi demo.
2.	Đảm bảo đầy đủ chức năng theo đúng đề tài
-	Web bán hàng: thêm giỏ hàng, thanh toán, quản lý sản phẩm…
-	Đặt lịch: quản lý lịch, đặt, hủy lịch, thông báo… - 	AI OCR: upload ảnh, trích xuất thông tin chính xác.
-	Hệ thống quản lý: CRUD đầy đủ, thống kê, báo cáo.
-	Chatbot AI: trả lời nghiệp vụ liên quan đến website.
3.	Giao diện (UI/UX) rõ ràng, dễ sử dụng - 	Giao diện thân thiện, không lỗi bố cục.
-	Màu sắc, font chữ thống nhất.
-	Responsive (tương thích máy tính - điện thoại) với các đề tài Website.
4.	Hệ thống quản trị (Admin Panel) bắt buộc phải có
-	Quản lý tài khoản người dùng
-	Quản lý dữ liệu chính (sản phẩm, dịch vụ, lịch khám, phòng trọ…)
-	Theo dõi thống kê cơ bản (doanh thu, lượt đặt lịch, số lượng người dùng…)
5.	Kiến trúc hệ thống rõ ràng, mã nguồn sạch - 	Áp dụng mô hình kiến trúc phù hợp (MVC).
-	Tổ chức file code, thư mục khoa học.
-	Có comment giải thích các phần quan trọng.
-	Không copy, paste tùy tiện.
6.	Thiết kế cơ sở dữ liệu đầy đủ và chuẩn hóa
-	ERD (Entity Relationship Diagram)
-	Các bảng thật trong hệ thống
-	Khóa chính, khóa ngoại, ràng buộc dữ liệu
-	Tránh trùng lặp dữ liệu
7.	Có tài liệu phân tích & thiết kế hệ thống
-	Use Case Diagram
-	Activity Diagram
-	Class Diagram
-	Sequence hoặc State Machine (nếu phù hợp)
-	Mô tả chức năng (SRS ở dạng đơn giản)
 8. Có kiểm thử phần mềm kèm minh chứng
Bắt buộc phải nộp:
-	Test Plan
-	Test Case (tối thiểu 15 - 30 test case)
-	Kết quả kiểm thử (Passed/Failed)
-	Bug Report (nếu có lỗi)
-	Với đề tài AI or OCR, cần có “bộ dữ liệu thử nghiệm” và “đánh giá độ chính xác”.
9.	Tài liệu báo cáo đúng cấu trúc học thuật - 	Bố cục đầy đủ các chương.
-	Trình bày khoa học, hình ảnh rõ nét.
-	Có tài liệu tham khảo đúng chuẩn.
10.	Tính bảo mật và hiệu năng ở mức cơ bản
-	Mật khẩu phải được mã hóa.
-	Phân quyền người dùng rõ ràng.
-	Kiểm tra lỗi input, tránh SQL Injection, XSS.
-	Hệ thống chạy mượt, không chậm hoặc treo.

11.	Yêu cầu chi tiết từng nhóm sản phẩm:
11.1. Nhóm Website bán hàng / mỹ phẩm / quần áo / phụ kiện / trái cây / đồ gia dụng
✓	Danh mục sản phẩm
✓	Lọc & tìm kiếm
✓	Giỏ hàng
✓	Đặt hàng
✓	Quản lý đơn hàng
✓	Quản trị sản phẩm – tồn kho
✓	Thanh toán (nếu có)
11.2. Nhóm Website đặt lịch / phòng khám / sân bóng / khách sạn
✓	Lịch trống, lịch đã đặt
✓	Đặt, hủy, sửa lịch
✓	Thông báo (email/SMS nếu có)
✓	Quản lý bác sĩ/sân bóng/phòng khách sạn
✓	Tự động gợi ý khung giờ (nếu tích hợp AI)
1.3. Nhóm có AI (Chatbot, OCR, Gợi ý sản phẩm, Dự đoán học vụ…)
✓	Bộ dữ liệu (dataset)
✓	Huấn luyện mô hình hoặc tích hợp API AI
✓	Độ chính xác (accuracy) phải có kết quả minh chứng
✓	So sánh mô hình / đánh giá hiệu năng
✓	Minh chứng đầu vào, đầu ra
1.4. Nhóm quản lý (học vụ, thư viện, phòng trọ, dữ liệu game…)
✓	CRUD đầy đủ
✓	Tìm kiếm – thống kê
✓	Phân quyền
✓	Báo cáo (PDF/Excel nếu có)
1.5. Nhóm kiểm thử phần mềm
(VD: Thiết kế và đánh giá test case Black-box cho sv.thanhdo.edu.vn)
✓	Test Plan
✓	Test Case EP/BVA/Decision Table/State Transition
✓	Kết quả kiểm thử
✓	Đánh giá độ bao phủ
✓	Bug Report
✓	Kiến nghị cải tiến hệ thống
